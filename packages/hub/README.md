# @nissegroup/hub

The Spakwus public web app — the **read plane**. A mobile-first, installable PWA
for the Sea to Sky corridor. A product of Nisse Group Ltd. Software only.

**Stack:** React · Vite · TypeScript · Tailwind · MapLibre GL (free, key-less OSM
raster style) · TanStack Query · vite-plugin-pwa (Workbox). It reads all brand
values (name, colours, logo, theme) from
[`@nissegroup/shared`](../shared/src/brand.ts) and talks to the
[`@nissegroup/core`](../core) read API.

Designed for a stopped (not driving) user: large targets, high-contrast status,
short glanceable copy.

## Pages

- **Home** — a large corridor status banner (green OPEN / amber PARTIAL / red
  CLOSED, with closure location + estimated reopening when available; state from
  the status pipeline only); a three-segment strip with per-segment state and
  last-updated; latest incident cards with a `source + timestamp + confidence`
  badge (`Official` / `Corroborated` / `Reported, unconfirmed`); live cam
  thumbnails with capture times; a "last updated" line and an honest coverage
  note. Each incident card reserves an (empty) slot for the Phase-4
  community-request count.
- **Map** — a MapLibre corridor map with incident markers (coloured by status)
  and camera markers. Code-split so MapLibre only loads here.
- **Incident detail** — the timeline of reports feeding one incident, each with
  provenance (source, confidence, steward flag, time).
- **About** — states plainly that Spakwus is best-available community information
  from Nisse Group Ltd, *not an official ruling*, and links DriveBC.

The **DriveBC** official-source link appears on every status view (banner,
coverage note, incident detail, About).

### Confidence, honestly

An `unconfirmed` report is styled so it can never be mistaken for an
authoritative closure: `Official` is a solid, dark, high-contrast badge;
`Reported, unconfirmed` is pale with a dashed outline. Community-board activity
never affects status.

## Offline behaviour

- Every successful `/api/status/snapshot` fetch is persisted to **IndexedDB**
  ([`offline/snapshotStore.ts`](src/offline/snapshotStore.ts)).
- When the API is unreachable **or slow** (the client aborts after a timeout),
  the app renders the last cached snapshot behind a prominent
  *"Showing saved conditions. Last updated N minutes ago. You may be offline —
  this is not live."* banner. A stale snapshot is **never** shown as live
  ([`offline/useCorridor.ts`](src/offline/useCorridor.ts)).
- Workbox precaches the app shell and opportunistically caches corridor map tiles
  (OpenStreetMap) and DriveBC cam thumbnails, so a snapshot opened in town still
  renders after signal is lost.
- The app is installable (Web App Manifest + service worker); manifest name,
  theme colour and icons come from the brand config.

## Branding

App title, manifest name, theme colour, footer, and the header logo placeholder
all read from the shared brand config. Colour/logo values there are marked-`TODO`
placeholders. Placeholder PWA icons are generated by
[`scripts/gen-icons.mjs`](scripts/gen-icons.mjs) (`pnpm --filter @nissegroup/hub
gen:icons`) — replace with the final brand assets when available.

## Running

```bash
# From the repo root, with @nissegroup/core running on :3000
pnpm dev:hub          # Vite dev server on :5173, proxies /api -> :3000
pnpm build:hub        # production build (PWA service worker generated)
pnpm --filter @nissegroup/hub run preview   # preview the production build
```

The service worker is disabled in dev; to exercise install/offline, use the
production build via `preview`.
